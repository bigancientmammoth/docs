

# Общая информация

Здесь описывается, как настроить подключение вашего ИИ (Claude, ChatGPT, Gemini и др).
Описание будет на примере Anthropic Claude, тем не менее все современные ИИ агенты поддерживают протокол [MCP](https://modelcontextprotocol.io/docs/getting-started/intro), то есть могут подключаться через платформу Marketaut к вашему магазину ВБ.


# Подключение

Запустите приложение Claude for Desktop

Нажмите на свой профиль в левом нижнем углу

Откройте **Settings**

![](img/c-open-settimgs.png)

Перейдите в раздел **Developer**
![](img/c-open-developer.png)

Нажмите **Edit Config**
![](img/c-open-config.png)


Выберите файл **claude_desktop_config** и откройте его в любом текстовом редакторе

В открывшемся файле найдите блок `mcpServers`

Добавьте в него описание ранее созданного MCP сервера Marketaut как показано ниже (можно скопировать)

```
  "mcpServers": {
    "marketaut": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://marketaut.ru/ebus/ep/v1/mcp",
        "--header",
        "Authorization:ВАШ_КЛЮЧ"
      ]
    }
  },
```

Перезапустите Claude. Необходимо не просто закрыть окно, а выйти (File->Exit) и зайти заново.

# Категории tools

Tools могут быть разделены на категории.

Например:
- `Read-only tools`
![c-readonly-tools](c-readonly-tools.png)
- `Write/delete tools`
![c-write-delete-tools](c-write-delete-tools.png)

---

## Read-only tools

Tools только для чтения данных.

Примеры:
- Цены товаров
- Категории
- Предметы
- Цвета
- Характеристики
- Карточки товаров

---

## Write/delete tools

Tools для изменения данных.

Примеры:
- Создать карточку
- Обновить карточку
- Установка цены
- Добавить фото
- Переместить карточку в корзину

---

# Управление доступом к tools

Справа возле каждого tool отображаются кнопки доступа.
![c-tool-mode](c-tool-mode.png)
## Always allow

`Always allow`

Tool полностью разрешен.

Claude сможет использовать его автоматически без подтверждения.

---

## Needs approval

`Needs approval`

Claude будет запрашивать подтверждение перед использованием tool.

---

## Blocked

`Blocked`

Tool отключен.

Claude не сможет использовать конкретный tool.

---

# Общая настройка категории

Для всей категории tools можно выбрать режим доступа сверху справа.

Например:
- `Always allow`
- `Custom`

---

## Always allow

Все tools внутри категории будут автоматически разрешены.

---

## Custom

Можно вручную настроить доступ отдельно для каждого tool.

---

# Отключение отдельного tool

Если необходимо отключить только определенный tool:

1. Откройте `Settings`
2. Перейдите в `Connectors`
3. Откройте `Customize`
4. Выберите нужный tool
5. Нажмите значок `Blocked`

После этого Claude не сможет использовать только выбранный tool.

Остальные tools продолжат работать.

---

# Полное отключение tools

Для полного отключения всех tools необходимо остановить диаграмму в `Приложении`.

1. Откройте `Приложение`
2. Перейдите в раздел `Конфигурация диаграмм`
3. Откройте нужную диаграмму
4. Нажмите кнопку `Stop`

После остановки диаграммы все tools перестанут работать в `Claude`.

---

# Что означает статус Stopped

Статус `Stopped` означает, что:
- диаграмма остановлена
- tools временно недоступны
- Claude не сможет выполнять действия через MCP
- API запросы перестают работать

Например:
- получение товаров будет недоступно
- работа с отзывами остановится
- tools перестанут отвечать

Для восстановления работы необходимо снова запустить диаграмму.

---

# Включение tools обратно

## Если был отключен отдельный tool

Необходимо снова разрешить его через:
- `Always allow`
или
- `Needs approval`

---

## Если была остановлена диаграмма

1. Откройте нужную диаграмму в `Приложении`
2. Нажмите кнопку `Play`

После запуска диаграммы tools снова станут доступны в `Claude`.

---

# Почему Claude может отвечать без tools

Claude не всегда использует tools автоматически.

Если запрос не требует получения данных через MCP, Claude может ответить самостоятельно без вызова tool.

Например:
- объяснение функций
- генерация текста
- помощь с документацией
- общие вопросы

Если необходимо использование tool, лучше формулировать запрос более конкретно.

Примеры:
- `покажи мои товары`
- `получи цены товаров`
- `создай карточку`
- `обнови остатки`

---

# Почему Claude может не использовать tool

Claude может не использовать tool если:
- tool отключен через `Blocked`
- диаграмма остановлена
- MCP не подключен
- отсутствует статус `Running`
- tool не соединен с `MCPv2`
- у блока не выбран аккаунт
- token указан неверно

---

# Как понять что Claude использовал tool

Если Claude использовал tool:
- отображается вызов tool
- может отображаться статус выполнения
- ответ содержит актуальные данные из MCP

Например:
- список товаров
- цены
- остатки
- данные карточек

---

# Как понять что tool не сработал

Признаки:
- Claude отвечает без актуальных данных
- появляется ошибка
- отсутствует вызов tool
- MCP не отображается со статусом `Running`

---

# Что делать если tools не отображаются

1. Проверьте что диаграмма запущена
2. Проверьте статус `Running`
3. Перезапустите `Claude`
4. Проверьте соединение blocks и `MCPv2`
5. Проверьте корректность token

---

# Что делать если MCP не отображается

Проверьте:
- запущена ли диаграмма
- корректно ли указан token
- соединены ли blocks с MCPv2
- выбран ли account у block
- перезапущен ли Claude

---

# Что делать если tools не работают

Возможные причины:
- diagram остановлена
- отсутствует статус `Running`
- tool заблокирован
- отсутствует подключение к MCP
- неверный token
- block не подключен к account

---

# Что происходит после подключения tools

После успешного подключения:
- Claude получает доступ к инструментам
- появляются MCP tools
- становятся доступны действия через API
- Claude может работать с подключенными сервисами

Например:
- получать товары
- изменять карточки
- работать с отзывами
- обновлять данные

---

# Важно

Для корректной работы:
- диаграмма должна быть запущена
- MCP должен отображаться со статусом `Running`
- token должен быть указан корректно
- tools внутри диаграммы должны быть настроены корректно
- blocks должны быть подключены к MCP
- у blocks должен быть выбран account

Claude использует только те tools, которые:
- подключены к MCP
- находятся в активной diagram
- имеют подключенный account
- разрешены в настройках Claude
