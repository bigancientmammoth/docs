Настройка диаграммы

Для настройки диаграммы:

  1. Перейдите в Конфигурация -> Диаграммы
  2. Откройте нужную диаграмму (кнопка с квадратом в строке диаграммы)

Пример окна настройки

(1) Наименование диаграммы и кнопка его редактирования
(2) Текущее и максимальное количество диаграмм в команде
(3) Версия диаграммы
(4) Идентификатор (ID) диаграммы
(5) Текущее состояние диаграммы
(6) Кнопка запуска и останова диаграммы
(7) Кнопка добавления нового элемента на диаграмму
(8) Область лога событий, связанных с диаграммой
(9) Холст с размещенными на нем узлами

Добавление MCP block

Как добавить MCP

  1. Нажмите кнопку:
    • Добавить блок

  2. В левой колонке выберите:

    • LLM

  3. В правой колонке выберите:

    • MCP v2

  4. Нажмите кнопку добавления

После добавления MCP block появится внутри рабочей области diagram.

После этого MCP block появится на рабочей области diagram.

Настройка MCP block

Подключение MCP

  1. Нажмите на MCP block

  2. Справа откроется панель управления

Необходимо заполнить поле:

ID

ID — это идентификатор, по которому MCP будет подключаться к Claude.

После внесения ID:

  1. Нажмите кнопку:

    • Сохранить

  2. Нажмите:

    • Закрыть

Изменения будут сохранены.

Добавление blocks Wildberries

Как добавить block Wildberries

  1. Нажмите кнопку:

    • Добавить блок

  2. В левой колонке выберите:

    • Wildberries

  3. Выберите нужный block

Например:
- каталог товаров
- отзывы
- вопросы
- аналитика

  1. Нажмите синюю кнопку:
    • Добавить

После этого block появится на diagram.

Подключение аккаунта к block

После добавления block на нем появится красная точка.

Это означает, что block не подключен к аккаунту.

Как подключить аккаунт

  1. Нажмите на нужный block

  2. Справа откроется панель управления

  3. В поле аккаунта выберите ранее созданный аккаунт

  4. Нажмите кнопку:

    • Сохранить

После подключения аккаунта красная точка исчезнет, а block станет готов к работе.

После этого аккаунт будет подключен к блоку.

Подключение blocks к MCP

Как соединить blocks

После настройки blocks необходимо подключить их к MCP block.

Для подключения:

  1. Проведите линии от block Wildberries к MCP block

  2. Подключите каждый необходимый раздел block

  3. Перетащите соединения мышкой между blocks

Типы подключений

Tools → Tools

Используется для подключения tools Wildberries к MCP.

Call → Call

Используется для подключения External Endpoint к MCP.

Важно

Если blocks подключены неправильно:
- MCP может не запускаться
- Claude не увидит tools
- часть функций работать не будет
- diagram может отображать ошибки

После подключения blocks смогут работать через MCP и Claude.

Если при подключении появилась ошибка, обновите страницу и повторите подключение.

Запуск diagram

После подключения всех необходимых blocks diagram готова к работе.

Как запустить diagram

  1. Нажмите кнопку:
    • Play

Кнопка находится в верхнем левом углу.

  1. Дождитесь запуска diagram

После успешного запуска:
- MCP начнет работать
- Claude сможет использовать tools
- подключенные blocks станут активны

Важно

Если diagram не запущена:
- MCP работать не будет
- Claude не увидит tools
- часть функций станет недоступна

Остановка diagram

Как остановить diagram

Для остановки diagram нажмите кнопку:
- Stop

Кнопка находится в верхнем левом углу.

После остановки:
- MCP перестанет работать
- Claude потеряет доступ к tools
- подключенные blocks станут недоступны до повторного запуска diagram

Готовность diagram

После:
- настройки MCP
- подключения аккаунтов
- подключения blocks
- соединения blocks с MCP
- запуска diagram

система будет готова к работе через Claude.

После запуска:
- Claude сможет использовать tools
- blocks начнут выполнять запросы
- MCP станет доступен для подключения
- diagram перейдет в рабочее состояние

Дополнительные блоки для diagram

Что означает статус Не сконфигурирована

Статус Не сконфигурирована означает, что diagram еще не готова к работе.

Например:
- не подключен MCP
- не подключены blocks
- отсутствует аккаунт
- blocks не соединены между собой
- diagram еще не была запущена


После полной настройки и запуска diagram будет готова к работе.

Diagram остановлена

Статус Остановлена означает, что diagram сейчас не активна и подключенные blocks временно не работают.

Для восстановления работы необходимо повторно запустить diagram через кнопку:
- Play

Во время остановки:
- Claude не сможет использовать подключенные tools
- MCP перестает обрабатывать requests
- blocks Wildberries не будут выполнять действия
- API requests временно не работают
- подключенные функции становятся недоступны

Например:
- Claude не сможет получать данные товаров
- работа с отзывами будет недоступна
- аналитика не сможет выполняться
- tools внутри diagram перестанут отвечать

Когда diagram может быть остановлена

Diagram может быть остановлена:
- вручную пользователем
- после изменений в настройках
- при ошибке подключения
- после перезапуска системы

Как возобновить работу

Для повторного запуска diagram:

  1. Нажмите кнопку:

    • Play

  2. Дождитесь статуса:

    • Запущена

После запуска:
- blocks снова станут активны
- Claude получит доступ к tools
- diagram продолжит работу

Что означает статус Запущена

Статус Запущена означает, что:
- diagram активна
- MCP работает
- blocks подключены
- Claude может использовать tools
- API requests доступны

Во время работы diagram Claude получает доступ к подключенным функциям и сервисам.

Что будет если удалить block

При удалении block:
- соединения с MCP будут удалены
- Claude потеряет доступ к функциям block
- данные внутри block могут перестать обновляться

После удаления может потребоваться повторная настройка diagram.

Что означает версия diagram

Версия diagram показывает количество изменений и сохранений diagram.

После:
- изменения blocks
- изменения подключений
- обновления настроек

версия diagram может изменяться автоматически.

Что означает ID diagram

ID diagram — это уникальный идентификатор diagram внутри системы.

Он используется для:
- внутренней работы системы
- идентификации diagram
- работы сервисов и подключений

Что означает Used / Использовано

Параметр Использовано показывает количество подключенных blocks внутри diagram.

Например:
- 2/20 означает:
- используется 2 blocks
- максимально доступно 20

Как перемещать blocks

Blocks можно перемещать внутри рабочей области diagram.

Для перемещения:

  1. Зажмите block мышкой

  2. Перетащите его в нужное место

Это помогает удобно организовывать diagram.

Рекомендации по работе с diagram

Полезные советы

  • Используйте понятные названия diagram
  • Не удаляйте MCP block после настройки
  • После изменений проверяйте соединения
  • Перед запуском убедитесь, что все blocks подключены
  • Сохраняйте аккаунты перед настройкой blocks
  • При ошибках подключения обновите страницу

Работа Claude через diagram

Claude использует только те tools, которые:
- подключены к diagram
- соединены с MCP
- имеют подключенный аккаунт
- находятся в запущенной diagram

Если один из этапов отсутствует, часть функций может не работать.

Что означает блокировка на block

Иконка замка на block означает:
- block защищен
- настройки заблокированы
- block нельзя случайно изменить

Функция помогает избежать случайных изменений diagram.

Что делать после изменения diagram

После:
- добавления blocks
- удаления соединений
- изменения аккаунтов
- обновления MCP

рекомендуется:

  1. Сохранить изменения

  2. Проверить соединения

  3. Перезапустить diagram

Это помогает корректно применить настройки.

Что такое MCP

MCP — это основной block подключения Claude к tools и сервисам.

Через MCP Claude получает доступ:
- к blocks
- к API
- к данным сервисов
- к функциям Wildberries

Без MCP работа diagram невозможна.

Что такое blocks

Blocks — это отдельные tools внутри diagram.

Каждый block отвечает за свою функцию.

Например:
- каталог товаров
- отзывы
- аналитика
- вопросы
- заказы

Blocks можно подключать, настраивать и соединять между собой.

# Что означает красная точка на block

Красная точка означает, что block требует настройки.

Например:
- не подключен аккаунт
- отсутствуют данные
- block не настроен полностью
- отсутствуют необходимые подключения
- block не соединен с MCP

После правильного подключения аккаунта и сохранения настроек ошибка исчезнет.

# Что означает красный крестик


Красный крестик на соединении означает проблему подключения между blocks.

Чаще всего это связано с:
- ошибкой соединения
- временной ошибкой интерфейса
- некорректным подключением линии
- неполным подключением block

Как исправить проблему

Способ 1 — Обновить страницу

В большинстве случаев достаточно обновить страницу и повторить подключение.

После обновления:

  1. Откройте diagram снова

  2. Проверьте соединения blocks

  3. При необходимости подключите линии повторно

Способ 2 — Переподключить соединение

Если ошибка осталась:

  1. Удалите проблемное соединение

  2. Повторно проведите линию между blocks

  3. Проверьте подключение к MCP block

После повторного подключения ошибка обычно исчезает.

Важно

Все необходимые blocks должны быть подключены к MCP.

Если один из blocks не подключен или подключен неправильно, diagram может работать некорректно.

Рекомендации по работе с diagram

Полезные рекомендации

  • Перед настройкой diagram сначала создайте аккаунт
  • Сохраняйте изменения после настройки каждого block
  • Проверяйте подключение всех blocks к MCP
  • Не удаляйте MCP block после настройки
  • Если blocks не соединяются — обновите страницу
  • После изменений рекомендуется перезапустить diagram

Возможные ошибки

Diagram не запускается

Проверьте:
- подключен ли MCP
- подключены ли аккаунты
- соединены ли blocks
- заполнен ли ID у MCP block

Работа diagram в Claude

После запуска diagram Claude сможет:
- использовать подключенные blocks
- работать с API
- выполнять действия через tools
- получать данные сервисов

Именно через diagram Claude получает доступ к подключенным функциям.

Подключение нескольких blocks

В одну diagram можно добавить несколько blocks.

Например:
- каталог товаров
- отзывы
- аналитика
- вопросы

Все blocks могут одновременно работать через один MCP block.

Изменение diagram

Diagram можно изменять после создания.

Например:
- добавлять новые blocks
- удалять blocks
- менять подключения
- подключать новые аккаунты
- изменять настройки MCP

После изменений рекомендуется повторно запустить diagram.

Удаление diagram

Данный раздел будет дополнен позже.

В будущем здесь будет доступна информация:
- как удалить diagram
- что произойдет после удаления
- удаляются ли подключенные blocks
- удаляются ли аккаунты вместе с diagram
- можно ли восстановить diagram после удаления

Важно

- остановить diagram
- проверить подключенные blocks
- сохранить необходимые настройки
- убедиться, что diagram больше не используется
