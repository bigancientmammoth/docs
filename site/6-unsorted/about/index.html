Marketaut — это low-code платформа для подключения ИИ-инструментов.
Она позволяет визуально строить автоматизации без написания кода.

Платформа состоит из следующих основных концепций:

Команда

Команда - это общее рабочее пространство для нескольких пользователей.

Всё, что создаётся на платформе - аккаунты, диаграммы и другие ресурсы - хранится внутри Команды и доступно всем её участникам. Вы приглашаете нужных людей, и они сразу получают доступ к общим материалам.

Это удобно для совместной работы: не нужно передавать файлы или дублировать настройки - всё уже в одном месте.

Диаграмма

Диаграммы - визуальные схемы, на которых строится логика автоматизации. На ней вы размещаете модули на холсте и соединяете их между собой, выстраивая нужную логику работы.

Одна диаграмма - одна автоматизация. Например, подключение вашего магазина Wildberries к MCP-серверу.

Все диаграммы хранятся внутри Команды и доступны её участникам.

Аккаунт

Аккаунты - это сохранённые учётные данные для подключения к внешнему сервису. Например, токен API от магазина Wildberries, логин и пароль от другого сервиса.

Вы настраиваете аккаунт один раз, а затем используете его в любых модулях, которым нужен доступ к этому сервису. Не нужно вводить одни и те же данные повторно в каждом модуле.

Все аккаунты хранятся внутри Команды и доступны её участникам

Принцип работы

Диаграмма — это холст, на котором вы размещаете узлы и соединяете их между собой.

Узел — это отдельный блок с конкретной функцией (например, «WB Контент», «MCP», «получить данные из CRM»). Узлы общаются между собой через связи — точки входа и выхода.

После настройки Диаграмму следует "Запустить". В запущенном режиме Диаграмма начинает обрабатывать все события. Любое существенное измение на Диаграмме (например, установка свойства узла) автоматически ее останавливает и требует последующего повторного запуска.

Типы коннекторов

У каждого модуля есть четыре типа коннекторов:

Асинхронные (потоковые) — для событий

Коннектор Роль Как работает
Stream Input Вход для событий Модуль получает событие, обрабатывает его и подтверждает получение
Stream Output Выход для событий Модуль отправляет событие — либо в ответ на входящее, либо самостоятельно (как триггер)

Пример: пришло письмо → модуль-триггер отправляет событие → следующий модуль его обрабатывает.

Синхронные (запрос-ответ) — для функций и данных

Коннектор Роль Аналог
Data Output Экспортирует функцию или данные наружу HTTP-сервер
Data Input Запрашивает функцию или данные у другого модуля HTTP-клиент

Data Output — модуль объявляет, что он умеет делать, и описывает формат запроса и ответа. Например, модуль может экспортировать MCP Tool, чтобы им мог воспользоваться другой модуль.

Data Input — модуль говорит «мне нужен какой-то внешний инструмент с таким-то форматом». К нему позже подключают подходящий Data Output. При этом модулю не важно, что именно будет подключено — главное, чтобы формат совпадал.

Как соединяются модули?

На диаграмме вы соединяете коннекторы линиями. Доступно два типа соединений:

Stream Output → Stream Input Передаёт события от одного модуля к другому. Одно событие можно направить сразу в несколько модулей — каждый получит свою копию.

Data Output → Data Input Предоставляет функцию одного модуля другому. Так модули «одалживают» друг другу возможности.

Модули

marketaut - это low code платформа с pluggable modules.

Каждый модуль реализует определенный функционал через javascript.

У модуля есть следующие коннекторы:

** Stream Input **

Асинхронный событийный входной канал. На него приходят события, которые модуль должен обработать.
После обработки событие квитируется (acknowledge).

** Stream Output **

Асинхронный событийный выходной канал.
Модуль может инициировать новые события как в результате обработки входных (Stream Inputs), так и самостоятельно (triggers).

** Data Output **

Синхронный сервер, аналог классического HTTP сервера.
Data Output - это способ предоставления функционала.
Предполагает запрос-ответ.
Когда модуль "экспортирует" какой-то функционал (например, MCP Tool), он объявляет его как Data Output и дополняет
информацией о структуре запроса и ответа (requestModel, responseModel).
Внешний модуль обращается к Data Output и получает ответ на свое обращение.

** Data Input **

Синхронный клиент, аналог классического HTTP клиента.
Data Input - это способ получения функционала.
Когда модулю нужны какие-то внешние инструменты, он объявляет один или несколько Data Input, к которым позднее будут подключены Data Output.
Сам модуль не знает, что к нему будет подключено, он лишь декларирует ожидаемую структуру запроса и ответа.

Marketaut позволяет визуально размещать модули на Диаграмме (холсте) и соединять их между собой, настраивая
потоки данных.

Соединять можно:

Stream Output -> Stream Input: обеспечивает передачу события с выходного одного модуля на вход(ы) другого(других).
Одно событие может быть отправлено на несколько входов (будут созданы копии).

Data Output -> Data Input: предоставление функционала одних модулей другим
