# Правила

## Именование файлов

1. Только **строчные** символы. ЗАГЛАВНЫЕ использовать нельзя. Причина: различие в интерпретации между разными ОС (Windows / MacOS)
2. Только английский буквы, дефисы `-` и подчеркивания `_`

## Контент

Списки оформляются через цифру с точкой. 

Например `1.` 

Варианты вроде `1)` использовать НЕЛЬЗЯ. Причина: различные парсеры Markdown по-разному интерпретируют списки, а стандарт предписывает точку.

## Внутренние ссылки

Внутренняя ссылка (fragment) - это ссылка на раздел внутри страницы. 

Стандартная markdown ссылка выглядит так: `[Лала](#header)`. Проблема в том, что заголовки на кириллице преобразуются
сборщиком в html-теги вида `<h3 id="_1"/>`, где от самого заголовка - только `_1`, причем эта цифра меняется при изменении
структуры страниц.

Решение: в md-файл **перед** теми местами, куда нужна ссылка (читай - перед заголовком) ставится незаметный html-якорь 
вида `<h5 id="ref_whatever"></h5>`. И ссылка делается на него, а не на идущий следом заголовок: `[Лала](#ref_whatever)`
