
# Руководство по быстрому старту

Все примеры, представленные ниже, предполагают, что вы уже зарегистрировались и вошли на платформу.

Если вы еще не зарегистрировались или не вошли, следуйте инструкциям в разделе [Регистрация](../3-start/register.md)


## Подключение Claude к магазину Wildberries


### Выписываем токен Wildberries API

Любая интеграция с Wildberries начинается с создания токена API. Он позволяет настроить получение данных из ВБ магазина

> [Здесь](../4-platform/02-account-types.md) рассказывается подробнее про интеграцию с Wildberries. 

> ⚠️ **ВНИМАНИЕ:** Вы должны быть владельцем ЛК, чтобы выписать токен

Зайдите на [Портал Продавца ВБ](https://seller.wildberries.ru), перейдите в раздел [Интеграции по API](https://seller.wildberries.ru/api-integrations) и нажмите **+ Создать токен**. 

В открывшемся окне выберите **Для интеграции вручную**, далее **Базовый токен**. 

Введите название для токена (на ваше усмотрение) и выберите разделы, к которым он будет иметь доступ. 

Для работы всех функций Marketaut следует выбирать **Контент**, **Цены и скидки** и **Документы**.

Выбрав разделы, нажимаете **Создать токен** и копируете полученную строку в буфер (`Ctrl+C` или `Command+C`)

![](img/portal-new-token.png)


### Создаем Аккаунт для работы с ВБ

Теперь надо создать Аккаунт, то есть учетную запись для работы с Wildberries, хранящую токены.

> Подробнее про аккаунты можно прочитать в разделе [Аккаунты](../4-platform/01-accounts.md)

Заходим на MarketAut в раздел [Конфигурация->Аккаунты](https://marketaut.ru/app/config/accounts)

Нажимаем кнопку **Добавить Аккаунт**

Выбираем Тип Аккаунта - **Wildberries**. Заполняем поля:

**Название**: *Магазин* (можно написать любое название, оно только для вас)
**Описание**: *Оставляем пустым* (можно написать любой текст, оно только для вас)
**Токен API**: Вставляем токен API, полученный в ЛК продавца Wildberries на предыдущем шаге.

![](img/new-account.png)

Жмем **Сохранить** и видим плитку только что созданного аккаунта. 

![](img/created-account.png)
На картинке синим цветом отображаются те разделы API, к которым вы предоставили доступ, создавая токен.

### Создаем диаграмму с агентом

Теперь необходимо создать Диаграмму, на которой настроить, к каким инструментам будет иметь доступ ИИ.

> Подробнее про диаграммы можно прочитать в разделах [Диаграммы](../4-platform/03-diagrams.md) и [Настройка диаграммы](../4-platform/04-diagrams-edit.md)

Заходим на MarketAut в раздел [Конфигурация->Диаграммы](https://marketaut.ru/app/config/diagrams)

Нажимаем **Добавить диаграмму**.

Вводим:
**Название**: *Агент по товарам* (можно написать любое название, оно только для вас)
**Описание**: *Оставляем пустым* (можно написать любой текст, оно только для вас)

![](img/new-diagram.png)

Жмем **Сохранить**. 

В окне со списком Диаграмм появилась новая диаграмма со статусом **Не сконфигурирована**

Жмем кнопку открытия диаграммы (квадрат)

![](img/open-diagram.png)

Перед нами - пустое полотно, на котором нужно разместить узлы

### Настройка диаграммы

Добавим на диаграмму узлы для магазина Wildberries и Агента.

> Подробнее можно прочитать в разделе [Настройка диаграммы](../4-platform/04-diagrams-edit.md)

На холсте жмем кнопку добавления узла:

![](img/add-node.png)

В списке выбираем **Каталог товаров** и жмем **Добавить**. 

![](img/add-content.png)

Мы создали узел для работы к контентом Wildberries. 
Теперь для этого узла необходимо указать Аккаунт, то есть с каким реальным магазином ему следует работать. 
Вы можете настроить работу сразу с несколькими магазинами, например, для сравнения их показателей. Каждый магазин - это один Аккаунт.

Для привязки к Аккаунту нажмите на созданном узле и в открывшейся справа "шторке" выберите закладку Аккаунты.

В выпадающем списке выберите свой аккаунт и нажмите **Сохранить**.

![](img/set-account.png)



Теперь повторяем процесс и добавляем узел **MCP**. Это узел для связи с вашим ИИ.
![](img/add-mcp.png)

Узлу MCP нужно указать свойство **endpoint**. Это внешнее имя, под которым вы сможете подключить данный узел в своего ИИ агента.

Вы можете создать несколько узлов MCP, дав каждому разное значение **endpoint**, например `content`, `legal`. Это позволит подключать узлы независимо к ИИ.

Выберите узел **MCP** на полотне и в открывшейся шторке на закладке **Свойства** введите значение `wb` и нажмите **Сохранить**.

![](img/set-property.png)


Узлы настроены, остается только решить, какие инструменты по работе с контентом вы хотите сделать доступными для ИИ.

Каждый выход (квадрат с подписью) у узлу "Каталог товаров" - это один инструмент. Например, `cards` - это чтение карточек товаров. `createCard` - это создание новой КТ. `updateCard` - изменение существующей КТ.

Вы можете выбрать любой набор, в рамках данного примера мы дадим ИИ доступ к чтению списка всех карточек `cards` и чтению одной карточки `getCardByNm`.

Протяните мышкой интересующие инструменты ко входу `tools` узла **MCP**.

![](img/connected.png)

ГОТОВО! 

Теперь запустите диаграмму. Запуск - это перевод диаграммы из режима "разработки", когда вы ее настраиваете, в режим работы.

![](img/start.png)


### Подключение ИИ

После того, как диаграмма настроена и запущена, остается подключить ее к ИИ. Мы будем делать это на примере **Claude**. 

> Подробнее о процессе подключения можно прочитать в разделе [Подключение Claude](../5-claude/01-claude-connect.md)

Мы не будем здесь подробно описывать процесс подключения, который описан в отдельном разделе (ссылка выше).

⚠️ **ВАЖНО**

Обратите внимание, что при подключении **Claude** (и любого другого ИИ) вам понадобится ссылка для подключения. Эта ссылка зависит от настроек узла **MCP**, сделанных ранее. Когда вы вводили свойство `endpoint`, то его значение стало частью ссылки.

Если вы ввели значение `wb`, то ссылка будет https://marketaut.ru/mcp/wb

Если вы ввели другое значение, то и ссылка будет другая. Вы всегда можете посмотреть точное значение ссылки, щелкнув на диаграмме на узле **MCP**:

![](img/copy-url.png)


Это значение следует использовать, добавля коннектор в Claude:

![](img/claude-connector.png)


После подключения вы увидите инструменты `cards` и `getCardByNm` в списке доступных для Claude:

![](img/claude-tools.png)
